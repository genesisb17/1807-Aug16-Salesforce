However, after our first review I managed to fumble AJAX fairly badly. Since then I have worked to ensure it will NOT happen again.
AJAX is of course Asynchronous JavaScript And XML, though it is not limited strictly to XML. It is a process used to communicate asynchronously beyween a client and server, and therefore allows the client to continue to operate rather than waiting on the server to respond.

The process is iteself simple, starting with the creation of a new XMLHttpRequest, followed by a callback function. This callback function ensures that the ready state and status are good before executing the function. The ready state must be 4 in order to proceed, this means it has gone through states 0 through 3 first. The status meanwhile is required to be 200 or 201, which means there has been a successful connection with the server.

Once these first steps are complete you then must use the open method of the XMLHttpRequest object, providing the following parameters: The method used to communicate with the server, if it is GET or PUSH, the URL to which the connection is being made, and if the communication is to be asynchronous. Optionally you might also have username and password parameters.

Last but not least you will use the XMLHttpRequest object's send method. If you are pushing data to the server you will pass the request body as a parameter. Optionally you might also set a request header prior to using the send method.

xhr = new XMLHttpRequest();
xhr.onreadystatechange(function{
if (xhr.readyState == 4 && xhr.status == 200){ Code to be executed }
});
xhr.open(METHOD, URL, ASYNC(True/False), Username(Optional), Password(optional));
xhr.setRequestHeader(header, value);
xhr.send(Request Body;)
